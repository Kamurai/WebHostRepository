Codage Unicode su un serveur Win32

La base de données est créée pour utiliser le codage Unicode (UTF8) lors du stockage des données sur un serveur PostgreSQL 8.0 fonctionnant sur une machine Win32.

L'implémentation Win32 de PostgreSQL 8.0 n'est pas capable de trier les chaînes de caractères suivant des séquences logiques appropriés car le système d'exploitation n'offre pas de support suffisant pour cela.

Pour cette installation, il est recommandé d'utiliser un codage spécifique de la locale, d'exécuter le cluster sur un système d'exploitation différent ou de mettre à jour votre cluster de bases de données avec PostgreSQL 8.1 qui dispose d'un code supplémentaire pour corriger ce problème.
